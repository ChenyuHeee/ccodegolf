# 背包问题优化 (Knapsack Optimization)

## 关键词
动态规划, 背包问题, 空间优化, 滚动数组

## 难度
10/10 (地狱级别)

## 题面
给定 n 个物品和一个容量为 W 的背包。第 i 个物品的重量为 w_i，价值为 v_i。每个物品只能选择一次，求在不超过背包容量的情况下，能获得的最大总价值。

输入格式：
- 第一行包含两个整数 n 和 W (1 ≤ n ≤ 500, 1 ≤ W ≤ 10000)
- 接下来 n 行，每行两个整数 w_i 和 v_i (1 ≤ w_i ≤ W, 1 ≤ v_i ≤ 1000)

输出格式：
- 输出一个整数，表示能获得的最大总价值

## 示范样例

### 样例输入 1
```
3 10
5 10
4 40
6 30
```

### 样例输出 1
```
70
```

### 样例输入 2
```
4 7
1 1
3 4
4 5
5 7
```

### 样例输出 2
```
9
```

### 样例输入 3
```
5 15
12 4
2 2
1 2
1 1
4 10
```

### 样例输出 3
```
15
```

## 提示
经典的 0-1 背包问题。标准动态规划解法需要 O(nW) 的空间，但可以使用滚动数组优化到 O(W)。在 code golf 中，关键是如何用最短的代码实现状态转移方程。
